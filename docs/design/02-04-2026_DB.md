# Database selection and design

Going forward, the following features are envisioned for this application:

- Resume review using LLM:
  - Resume storage (MIME: pdf, docx, odt)
  - LLM chats
- Job scraping based on keywords:
  - Job storage with
    - Source, Company, Contact, Poster, Link, Content, Match
- Email status tracking
  - Email sending with
    - Source, Destination, Related Job, Date, Status
  - Email threads using SMTP server

Upon categorisation, a database for referral should fulfill following criterion:

- Relational data support with large tables
- Reliable
- Scalable object store

Postgres fulfills the relational data requirement
with battle-tested reliability and scalability.
A simple Go server with fs-based S3-compatible object storage and
Postgres backed metadata store fulfill all the database requirements
for referral right now.

This will keep the application stack simplistic:

- Dashboard
- Referrer API
- Postgres
- S3-compatible fs-based object storage server with
Postgres metadata store

Hence, choosing Postgres.
